# discord-proxy-rs

## Описание
Этот проект представляет собой прокси-библиотеку (`version.dll`) для десктопного клиента Discord. Библиотека служит мостом между клиентом Discord и внешними инструментами проксирования (Clash, V2Ray и другие). Она расширяет возможности Discord, позволяя ему работать с прокси-серверами, поддержка которых изначально ограничена или отсутствует.

## Назначение
Программа перехватывает запуск процессов Discord и внедряет настройки через динамически генерируемые PAC-скрипты. Это обеспечивает маршрутизацию трафика основного процесса и вторичных процессов через заданные SOCKS5 или HTTP туннели.

## Причины реимплементации на Rust
- **Безопасность памяти**: Исключено возникновение типичных ошибок переполнения буфера и утечек ресурсов, характерных для системных перехватчиков на C++ и Delphi.
- **Производительность**: Минимальные задержки при запуске и отсутствие влияния на общую скорость работы приложения.
- **Современная база**: Использование эффективных инструментов для низкоуровневого перехвата функций и работы с WinAPI.
- **Масштабируемость**: Простая поддержка многопроцессной архитектуры Electron и предсказуемое поведение при установке хуков.
- **Чистота кода**: Модульная структура упрощает аудит и модификацию проекта сторонними разработчиками.

## Преимущества реализации через PAC
По сравнению с методами перехвата на уровне сокетов (использовавшимися в ранних реализациях на Pascal/C++), подход через PAC предоставляет:
- **Нативная интеграция**: Используется встроенный движок Chromium, что гарантирует полную совместимость со всеми типами запросов Electron (HTTP, WebSockets и т.д.).
- **Целостность SSL/TLS**: Проксирование происходит на уровне приложения. Это исключает ошибки при работе с зашифрованным трафиком, характерные для низкоуровневых перехватчиков.
- **Гибкость правил**: PAC-скрипты позволяют реализовывать сложную логику маршрутизации (балансировка, исключения) без изменения бинарного кода хуков.
- **Стабильность**: Снижается риск конфликтов с антивирусным ПО и другими системными сетевыми инструментами.

## Установка
1. Скомпилируйте проект с помощью команды `cargo build`.
2. Скопируйте полученный файл `version.dll` и файл конфигурации `drover.toml` в следующую папку:
   `%localappdata%/Discord/app-<версия>/` (например, `app-1.0.9147`).
3. Убедитесь, что адрес и порт в `drover.toml` совпадают с параметрами вашего прокси-сервера (Clash, V2Ray и т.д.).
4. Перезапустите Discord.

## Планы по развитию
- **Автономная работа**: Реализация алгоритмов обхода DPI (фрагментация TLS Client Hello) для подключения без внешних прокси-клиентов.
- **Фрагментация UDP**: Улучшенная логика дробления пакетов для обеспечения стабильной голосовой связи в условиях активного подавления трафика.
- **DNS-over-HTTPS (DoH)**: Интегрированное защищенное разрешение имен для предотвращения подмены DNS-ответов.
- **Управление трафиком**: Расширенная фильтрация доменов и поддержка протокола SOCKS5 UDP Associate.

## Настройка
Параметры задаются в файле `drover.toml`.
- `main`: Основной адрес SOCKS5/HTTP прокси.
- `rules`: Исключения и правила для конкретных доменов.

## Конфигурация

Если в `drover.toml` вы оставите `fallback` пустым или вообще удалите эту строку, логика в `src/config.rs` сделает следующее:

- `fallback_pac` получит значение `"DIRECT"`.
- При генерации цепочки скрипт увидит, что `fallback` — это `"DIRECT"`, и пропустит его дублирование.
- В итоге PAC-скрипт вернет: `PROXY 127.0.0.1:1080; DIRECT`.
- Результат для Discord: Chromium сначала попробует ваш `main` прокси. Если он не отвечает (сервис выключен или порт закрыт), Discord мгновенно и незаметно для вас переключится на прямое соединение через ваш обычный системный интернет-канал.

Это самый надежный режим: прокси работает, когда он запущен, и не "ломает" Discord, когда он выключен.
